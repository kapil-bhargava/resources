üì¶ What is a Block?

A block is a chunk of code enclosed within { }. It is commonly used in:

  • if, else conditions
  • for, while loops
  • function definitions
  • Even standalone: { let x = 5; }
Example:
{
  let message = "Hello Block";
  console.log(message); // ‚úÖ Works here
}
console.log(message); // ‚ùå Error: message is not defined
Variables declared with let and const are block scoped.

üîç What is Scope?

Scope determines where a variable is accessible in your code.

  • Global Scope ‚Äî accessible anywhere
  • Function Scope ‚Äî accessible only inside a function
  • Block Scope ‚Äî accessible only inside a block { }
Example:
let a = 10; // global scope

function test() {
  let b = 20; // function scope
  if (true) {
    let c = 30; // block scope
    console.log(a, b, c); // ‚úÖ 10 20 30
  }
  console.log(c); // ‚ùå Error: c is not defined
}
test();

üåê Global Scope

Any variable declared outside all functions or blocks is in the global scope. It's accessible from anywhere in your JavaScript code.

Example:
let website = "zninfotech.com";

function showSite() {
  console.log(website); // ‚úÖ Access global variable
}
showSite();
Be cautious: modifying global variables can cause bugs if reused in many places.

üîÅ Function Scope

Variables declared with var, let, or const inside a function can only be accessed within that function.

Example:
function greet() {
  let name = "Student";
  console.log(name); // ‚úÖ Accessible
}
greet();
console.log(name); // ‚ùå Error: name is not defined

üß† Lexical Scope (Static Scope)

Lexical Scope means a function can access variables defined in its outer scope based on where it was defined ‚Äî not where it is called.

Example:
function outer() {
  let outerVar = "I'm outside!";
  
  function inner() {
    console.log(outerVar); // ‚úÖ Accessible due to lexical scope
  }

  inner();
}
outer();
Lexical scope is fixed during code writing, not during runtime.
