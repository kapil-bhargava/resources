üîç What is Dynamic Memory Allocation?

Dynamic memory allocation in C means allocating memory at **runtime** using pointers. It allows us to create variables or arrays when we don't know the size in advance.

üìå Why and When to Use

  • When array size is not known at compile time
  • To avoid wastage of memory
  • Used in data structures like Linked List, Stack, Tree, etc.

‚öôÔ∏è Dynamic Allocation Functions

  • malloc(): Allocates single block, contains garbage values
  • calloc(): Allocates multiple blocks, initializes with 0
  • realloc(): Resizes previously allocated memory
  • free(): Frees the allocated memory

üß™ Demo Code: Using malloc()

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;
    int n, i;

    printf("Enter number of elements: ");
    scanf("%d", &n);

    ptr = (int*) malloc(n * sizeof(int));  // Allocate memory

    if(ptr == NULL) {
        printf("Memory not allocated!");
        return 1;
    }

    printf("Enter %d values:\n", n);
    for(i = 0; i < n; i++) {
        scanf("%d", &ptr[i]);
    }

    printf("Values entered:\n");
    for(i = 0; i < n; i++) {
        printf("%d ", ptr[i]);
    }

    free(ptr);  // Free memory
    return 0;
}

üìò calloc() vs malloc()

int *arr1 = (int*) malloc(5 * sizeof(int));  // Garbage values
int *arr2 = (int*) calloc(5, sizeof(int));   // All values = 0

‚ôªÔ∏è realloc() Example

int *ptr = (int*) malloc(2 * sizeof(int));
ptr = (int*) realloc(ptr, 4 * sizeof(int)); // Resize to 4 ints

üö´ Always Free Memory!

Use free(ptr); when memory is no longer needed to avoid memory leaks.
