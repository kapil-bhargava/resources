Dart Typedefs

Learn what typedefs are and how to use them in Dart

1. What is a Typedef? (Programming Definition)

A typedef in Dart is a way to create a new name for a function type. It helps you give a meaningful name to a function signature (parameters and return type), making your code easier to read and maintain.

Think of it as creating a shortcut or nickname for a function's "shape" or "blueprint."

2. Real-World Example of Typedef

Imagine you have many different workers who all follow the same instructions for a task — like "paint a wall." Instead of explaining the task every time, you give that task a name, like "PainterTask." Now whenever someone says PainterTask, everyone knows exactly what type of task it is.

3. Important Terms

  • typedef: Keyword used to define a function type alias.
  • Function Signature: The list of parameters and return type a function has.
  • Alias: Another name for something.

4. How to Define and Use Typedefs in Dart

// Define a typedef for a function that takes two integers and returns an integer
typedef Operation = int Function(int a, int b);

// Use the typedef to declare functions matching the signature
int add(int a, int b) {
  return a + b;
}

int multiply(int a, int b) {
  return a * b;
}

void performOperation(Operation op, int x, int y) {
  print('Result: ${op(x, y)}');
}

void main() {
  performOperation(add, 5, 3);       // Output: Result: 8
  performOperation(multiply, 5, 3);  // Output: Result: 15
}

Explanation:

  • Operation is the typedef — a name for a function that takes two ints and returns an int.
  • add and multiply functions match this typedef.
  • performOperation takes a function of type Operation and calls it.

5. Why Use Typedefs?

  • Improves code readability by giving meaningful names to function types.
  • Makes it easier to change function signatures in one place.
  • Useful in APIs and callbacks where you pass functions as parameters.

6. Summary

Typedefs let you name function types, making your Dart code cleaner and easier to understand. They help especially when working with complex function parameters or callbacks.
