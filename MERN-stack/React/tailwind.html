ü§î What is Tailwind CSS?

üéØ Simple Definition

Tailwind CSS is a utility-first CSS framework. Instead of writing custom CSS, you use pre-built classes directly in your HTML to style elements.

‚ö° Why It's Popular

It's faster to write, easier to maintain, and creates consistent designs. You don't need to think of class names or switch between HTML and CSS files.

‚ùå Traditional CSS Way

/* CSS File */
.my-button {
  background-color: blue;
  color: white;
  padding: 12px 24px;
  border-radius: 8px;
  border: none;
}

/* HTML File */
<button class="my-button">
  Click me
</button>

‚úÖ Tailwind CSS Way

/* No separate CSS file needed! */

/* HTML File */
<button class="bg-blue-500 text-white px-6 py-3 rounded-lg border-0">
  Click me
</button>

üîß How Tailwind Classes Work

üéØ The Pattern: Property-Value

Tailwind classes follow a simple pattern. Each class represents a CSS property and its value:

üìù Class Structure Examples

text-red-500    ‚Üí    color: #ef4444;
bg-blue-600     ‚Üí    background-color: #2563eb;
p-4             ‚Üí    padding: 1rem;
m-2             ‚Üí    margin: 0.5rem;
w-full          ‚Üí    width: 100%;
h-screen        ‚Üí    height: 100vh;
üí° Pro Tip: The numbers (like 500, 600) represent intensity. Higher numbers = more intense (darker colors, larger sizes).

üé® Live Example

Here's a button styled with Tailwind classes:

<button class="bg-green-500 hover:bg-green-600 text-white font-bold py-2 px-4 rounded">
  Success Button
</button>

What each class does:

  • bg-green-500 - Green background
  • hover:bg-green-600 - Darker green on hover
  • text-white - White text
  • font-bold - Bold font weight
  • py-2 - Padding top/bottom
  • px-4 - Padding left/right
  • rounded - Rounded corners

üöÄ How to Use Tailwind Classes

üéØ Basic Usage Rules

1. üì± Responsive Design

Add screen size prefixes to make responsive designs:

<div class="text-sm md:text-lg lg:text-xl">
  Responsive text size
</div>

Breakpoints: sm (640px), md (768px), lg (1024px), xl (1280px)

2. üé≠ Hover & Focus States

Add state prefixes for interactive elements:

<button class="bg-blue-500 hover:bg-blue-700 focus:ring-4">
  Interactive Button
</button>

3. üåô Dark Mode

Add dark: prefix for dark mode styles:

<div class="bg-white dark:bg-gray-800 text-black dark:text-white">
  Dark mode ready!
</div>
