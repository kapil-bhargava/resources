1. node_modules/

Just like CRA, this folder stores all installed npm packages (like React, Vite plugins). Managed by npm or Yarn.

2. public/

Contains static files accessible at runtime. Vite serves files from this folder as-is without bundling via ESM.

3. index.html

The single HTML page where your React app is loaded. Vite injects the React app here automatically.

Vite HTML processing happens here.

4. src/ (Source Folder)

Main place for your app's code:

  • App.jsx: The root React component.
  • main.jsx: The entry point where ReactDOM.createRoot() renders your app to the DOM.
  • index.css: Global CSS styles applied throughout the app.
  • App.css: Styles specific to App.jsx.

5. vite.config.js

This file contains configuration options for Vite, like aliases, plugins, and server settings. Much simpler than Webpack config.

import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
});

6. Other Important Files

  • package.json: Lists dependencies and project scripts.
  • package-lock.json / yarn.lock: Locks package versions.
  • .gitignore: Tells Git to ignore files like node_modules.
  • README.md: Documentation for your app.

7. Vite vs CRA: What Changed?

  • No public/index.html separation â€” Vite uses index.html directly at root.
  • No complicated Webpack config â€” Vite simplifies this using vite.config.js.
  • Vite is designed for ESM and supports SSR (Server Side Rendering) easily.
Note: Vite is faster than CRA during development because it serves ESM files without bundling and transforms only changed files.

âœ¨ Important Terms Explained

  • Vite: Modern build tool replacing CRA for React projects.
  • ESM: Native module system supported in browsers and Node.js.
  • SSR: Used for SEO-friendly React apps.
  • Webpack: Previously the default tool CRA used (complex and slower).

Summary

  • Vite is lightweight, fast, and simple.
  • index.html is in the root (not in public/).
  • ESM and modern browser features are default.
  • No heavy Webpack config.
