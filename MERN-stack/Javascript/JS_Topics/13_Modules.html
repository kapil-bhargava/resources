JavaScript ES6 Modules - Explained in Detail

Modules allow you to split your JavaScript code into reusable files. Using import and export (introduced in ES6), you can manage code better, reduce file sizes, and keep your project organized.

Why Use Modules?

  • Reusability — share functions, classes, or variables across files.
  • Maintainability — keep code organized and easy to debug.
  • Scalability — manage larger projects by splitting logic into files.

Basic Syntax of Export

In file1.js:

// Named Export
export const name = 'Rahul';

export function greet() {
  return 'Hello from Module!';
}

// Default Export
const age = 25;
export default age;

Basic Syntax of Import

In file2.js:

import age, { name, greet } from './file1.js';

console.log(name);         // Output: Rahul
console.log(greet());     // Output: Hello from Module!
console.log(age);         // Output: 25

Types of Exports

  • Named Export: Export multiple items with names (using {} when importing).
  • Default Export: Export only one thing as the main exported value (import without {}).

Example: Splitting JS Code Using Modules

math.js (Module file)

// Exporting multiple named exports
export function add(x, y) {
  return x + y;
}

export function multiply(x, y) {
  return x * y;
}

app.js (Import file)

import { add, multiply } from './math.js';

console.log(add(2, 3));        // Output: 5
console.log(multiply(4, 5));   // Output: 20
Note: ES6 Modules only work when served via HTTP/HTTPS or local server (not directly opening HTML file in browser).

Use Case in Real Projects

  • Utility Functions: Create reusable utility functions like date formatting, calculations, etc.
  • Component-Based Apps: React, Vue, Angular all depend on modules for components, services, routing, etc.
  • API Services: Split API calling logic into service files.

Example: API Service Using Modules

apiService.js

export async function fetchData(url) {
  const response = await fetch(url);
  return await response.json();
}

main.js

import { fetchData } from './apiService.js';

fetchData('https://jsonplaceholder.typicode.com/posts/1')
  .then(data => console.log(data));

Conclusion

Modules in JavaScript improve structure, reusability, and scalability of code. ES6 import/export makes it easy to split code into meaningful files — a must for professional JavaScript development.
