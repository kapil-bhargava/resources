Understanding useContext in React

A complete guide to managing global state efficiently using Context API

What is useContext?

useContext is a React Hook that allows you to consume and subscribe to a React Context. It simplifies the process of accessing shared data across components without prop drilling.

What is Context API?

The Context API is a built-in API in React (introduced in v16.3+) that allows you to manage and share state globally. "API" means a set of tools (like createContext, Provider, useContext) provided by React to easily build and consume context.

Why use useContext?

  • Avoids prop drilling (no need to pass props manually through every level)
  • Makes global data like themes, user info, or settings available anywhere
  • Improves code readability and maintainability

Why is useContext better than Props?

  • Cleaner: Prevents messy deeply-nested props
  • Reusable: Makes shared state accessible across multiple components
  • Efficient: Only re-renders components that use the context

Steps to Use useContext

  1. Create Context with createContext()
  2. Provide context using Provider in parent
  3. Consume context using useContext() in any child

1. Create Context (ThemeContext.js)

import { createContext } from 'react';

const ThemeContext = createContext();
export default ThemeContext;

2. Provide Context (App.js)

import React from 'react';
import ThemeContext from './ThemeContext';
import Toolbar from './Toolbar';

function App() {
  const theme = { color: 'blue' };

  return (
    <ThemeContext.Provider value={theme}>
      <Toolbar />
    </ThemeContext.Provider>
  );
}

export default App;

3. Consume Context (Toolbar.js)

import React, { useContext } from 'react';
import ThemeContext from './ThemeContext';

function Toolbar() {
  const theme = useContext(ThemeContext);

  return (
    <h2 style={{ color: theme.color }}>
      Toolbar Text with Context Color
    </h2>
  );
}

export default Toolbar;

Common Use Cases

  • Dark/light theme toggling
  • User authentication status
  • Global app settings
  • Language preferences

Things to Avoid

  • Using context for deeply complex state (consider Redux)
  • Forgetting to wrap components with the Provider
  • Changing context value directly (use state)
  • Not using memoized context values (causes unnecessary re-renders)
