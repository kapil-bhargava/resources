Tracking Multiple Login Details using Node.js OS Module

Log and Send User System Info to Admin

üìå Why Use the os Module?

The os module is a built-in Node.js utility that helps fetch details about the operating system, user session, network interfaces, CPU, and memory. While it doesn't handle logins directly, it‚Äôs powerful when you want to capture user environment info when a login occurs.

‚úÖ Use Case: Logging Login Meta and Sending to Admin

You can gather and send the following details on each login:

  • Username on system
  • Device hostname
  • OS platform and architecture
  • Local IP address
  • Login time
üí° Example: When a user logs in, this metadata is sent to an admin dashboard or email for monitoring or audit purposes.

üì¶ Step-by-Step Code Example

// server.js
const os = require('os');
const axios = require('axios'); // npm install axios

function getLoginMetaData() {
  const user = os.userInfo();
  const device = os.hostname();
  const platform = os.platform();
  const arch = os.arch();
  const ip = Object.values(os.networkInterfaces())
    .flat()
    .find(net => net.family === 'IPv4' && !net.internal)?.address;

  return {
    user: user.username,
    device,
    platform,
    arch,
    ip,
    loginTime: new Date().toISOString()
  };
}

// Send login data to admin backend
async function sendLoginToAdmin() {
  const data = getLoginMetaData();
  try {
    await axios.post('https://your-admin-api.com/logins', data);
    console.log("Login data sent to admin successfully!");
  } catch (err) {
    console.error("Error sending login data:", err.message);
  }
}

// Trigger this function after user login
sendLoginToAdmin();

üß† Summary of Useful os Methods

  • os.userInfo() ‚Äì Get the current system username
  • os.hostname() ‚Äì Get the computer name/device
  • os.platform() ‚Äì OS type like 'win32', 'linux'
  • os.arch() ‚Äì CPU architecture (e.g., 'x64')
  • os.networkInterfaces() ‚Äì IP addresses and interface info

üö´ What the os Module Can't Do

  • It can't track actual user credentials or login events.
  • It can't send emails or store in database by itself ‚Äî you'll need external tools like Axios, Nodemailer, MongoDB.

üîê When to Use This?

  • To track suspicious login patterns
  • To build an admin panel showing all login devices/IPs
  • To monitor internal system logins (e.g., employee systems)
